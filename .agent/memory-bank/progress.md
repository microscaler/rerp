**2026-04-25 build analysis:** RERP build issues are centered on drift between suite-nested BRRTRouter support and the custom `rerp` wrapper. BRRTRouter raw discovery supports `openapi/{suite}/{service}` and `microservices/{suite}/{service}`; Hauliage's workspace CLI adds `<impl_package>_gen` naming and build metadata derivation. RERP had hardcoded `accounting`, generated some gen crate names as `rerp_accounting_*`, and mapped builds to package names that can be gen crates instead of impl crates (`general-ledger`, `accounts-receivable`). `rerp bff generate-system` also delegated to the raw BRRTRouter BFF command without passing the expected `openapi/accounting/openapi_bff.yaml` output, so the BFF spec could be stale/wrong. A local `cargo check -p rerp_accounting_general_ledger_impl` stalled in `aws-lc-sys` build-script before reaching RERP code and was stopped.
