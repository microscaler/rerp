# Accounting Service Impl Controller Generator Drift

- **Status**: `verified`
- **Source docs**: `microservices/accounting/*/impl/src/controllers/`, `microservices/accounting/*/gen/src/handlers/`, `openapi/accounting/*/openapi.yaml`
- **Code anchors**: `Tiltfile` (test-invoice, test-accounts-payable, etc.), `microservices/accounting/*/impl/Cargo.toml`
- **Last updated**: 2026-04-28

## What It Is

When accounting service OpenAPI specs were regenerated, the BRRTRouter generator produced new handler/type names in the `gen` crates. The existing `impl` controller stubs still imported the old handler names, causing compilation failures across 7 services.

## Problem

After regenerating gen code from updated OpenAPI specs:

- **39 orphaned impl controller files** across 6 services (invoice, accounts-payable, bank-sync, asset, budget, financial-reports) imported gen handlers that no longer existed.
- **41 orphaned `pub mod` entries** remained in each service's `controllers/mod.rs`, pointing to deleted files.
- **EDI gen type mismatch**: the generator produced `EdiFormat` where the impl expected `EdiStandard`, and `EdiSubmission` where the impl expected `EdiSubmissionStatus`. The struct `EdiSubmissionStatus` was also missing from the generated types module.

## Fix Applied

1. **Deleted 39 orphaned impl controllers**: Removed files in services that imported gen handlers no longer present after spec regeneration. These were stale stubs from previous OpenAPI revisions.
2. **Cleaned 41 `pub mod` entries**: Removed orphaned module declarations from `controllers/mod.rs` in all 6 affected services.
3. **Fixed EDI type names**: Corrected `EdiFormat` → `EdiStandard` and `EdiSubmission` → `EdiSubmissionStatus` type references across gen handlers, controllers, and impl imports.
4. **Added missing struct**: Added `pub struct EdiSubmissionStatus` to the generated `types.rs` so all field references compile.

## Gotchas

- **Never edit generated migration files directly** — update the single owning
  entity definition, then run RERP's suite-aware top-level migrator for
  Accounting. Service-owned entities live in
  `microservices/accounting/<service>/impl/src/models/`; genuine suite
  foundation entities live in `microservices/accounting/entities/src/`.
- **Impl controllers are per-spec** — when OpenAPI specs change, regenerated gen code will likely rename handlers/types. Old impl stubs become orphaned. Delete stale controllers rather than patching them to match new names.
- **Gen code is auto-generated** — files under
  `microservices/<suite>/<service>/gen/` are regenerated by BRRTRouter. Do not
  add manual definitions there. The proper fix for missing or mismatched types
  is the owning OpenAPI specification or BRRTRouter generator, not generated
  output.
- **Tilt caches test results** — after deleting files and cleaning `mod.rs`, Tilt may serve stale `Build Failed` logs. Touch files or force-trigger the test resource to clear the cache.
