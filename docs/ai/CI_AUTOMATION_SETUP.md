# CI Automation Setup - BFF Generation

## Overview

First CI automation for RERP: Automatic generation of system-level BFF OpenAPI specifications from sub-service specs.

## GitHub Actions Workflow

### Workflow File
- **Location**: `.github/workflows/generate-bff-specs.yml`
- **Purpose**: Auto-generate system BFF specs when sub-service specs change

### Triggers

The workflow runs on:
1. **Push to main/develop**: When OpenAPI specs change
2. **Pull Requests**: When OpenAPI specs are modified
3. **Manual Dispatch**: Can be triggered manually from GitHub Actions UI

### Path Filters

Only triggers when these paths change:
- `openapi/**/*.yaml` - Any OpenAPI spec file
- `scripts/generate_system_bff.py` - The generation script itself
- `.github/workflows/generate-bff-specs.yml` - The workflow file

### What It Does

1. **Checkout Code**: Gets the latest code including sub-service specs
2. **Setup Python**: Installs Python 3.12
3. **Install Dependencies**: Installs PyYAML from `scripts/requirements.txt`
4. **Generate BFF Specs**: Runs `scripts/generate_system_bff.py` to regenerate all system BFF specs
5. **Check for Changes**: Detects if any BFF specs were updated
6. **Commit or PR**:
   - **On Push**: Commits changes directly to the branch
   - **On PR**: Creates a new PR with the updated specs

### Output

- Updated system-level BFF specs in `openapi/{system}/openapi.yaml`
- Automatic commit message or PR description
- Summary in GitHub Actions logs

## Workflow Behavior

### On Push to Main/Develop

1. Detects changes to sub-service specs
2. Regenerates all system BFF specs
3. Commits changes with message:
   ```
   chore: auto-generate system BFF OpenAPI specs from sub-services
   
   Auto-generated by GitHub Actions workflow.
   This commit updates system-level BFF specs based on changes to sub-service OpenAPI specifications.
   ```

### On Pull Request

1. Detects changes to sub-service specs
2. Regenerates all system BFF specs
3. Creates a new PR branch: `auto-generate-bff-specs`
4. Opens PR with:
   - Title: "chore: Auto-generate System BFF OpenAPI Specs"
   - Labels: `automation`, `openapi`, `bff`
   - Description explaining the auto-generation

### No Changes

If BFF specs are already up-to-date:
- Workflow completes successfully
- No commits or PRs created
- Summary shows "No changes to BFF specs"

## Important Notes

⚠️ **Auto-Generated Files**
- System-level `openapi.yaml` files are auto-generated
- Manual edits will be overwritten
- Always edit sub-service specs, not BFF specs

✅ **Best Practices**
- Make changes to sub-service specs
- Let CI regenerate BFF specs automatically
- Review auto-generated PRs before merging
- Use descriptive commit messages for sub-service changes
